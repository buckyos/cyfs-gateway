# 能力边界与回答约束

本文件用于约束 skill 在回答“能不能做”“是不是等价于局域网”这类问题时的表达边界。

## 1. 回答分层

回答时要把结论分成三层：

- 已有明确证据支持的配置事实
- 可以据此给出的保守工程推断
- 当前 evidence 不足，不能下结论的部分

不要把第二层说成第一层，也不要把第三层补成“常识答案”。

## 2. 当前可直接确认的事实

根据 [implementation-checked.md](implementation-checked.md)，当前可以直接确认：

- `cyfs_gateway` 正式注册的 stack 协议包括 `tcp`、`udp`、`tls`、`quic`、`rtcp`、`tun`
- `tun` 配置包含 `bind`、`mask`、`mtu`、`tcp_timeout`、`udp_timeout`、`hook_point`
- `socks` 配置包含 `target`、`enable_tunnel`、`rule_config`、`hook_point`
- `socks` 会通过 `TunnelManager` 获取到 `proxy_target` 的 tunnel，再为请求目标打开流
- process chain 可用于路由、转发、命中规则和调用 server / global chain

这些属于“配置与执行模型事实”。

## 3. 关于 `tun` 的保守解释

在本 skill 当前证据范围内，讨论 `tun` 时应使用以下保守解释：

- `tun` 只能直接被理解为 IP 级虚拟接口相关配置
- 当前直接证据只覆盖 `bind`、`mask`、`mtu`、`tcp_timeout`、`udp_timeout`、`hook_point`
- 这些字段可用于解释地址与参数语义，但不能单独推出“已经形成可用私网 / 跨网段互通”
- 如果继续讨论跨设备互通，必须另外写出 underlay、路由、防火墙、NAT / MTU 和验证步骤，并明确这些属于部署前提，不是 `tun` 字段本身

## 4. 关于 `socks5` 的保守解释

在本 skill 当前证据范围内，讨论 `socks5` 时应使用以下保守解释：

- `socks5` 是应用代理接入模型
- 它适合“支持代理的 app 访问远端服务或远端网络资源”
- 它可以配合 `enable_tunnel` 和 `rule_config` 做按代理目标或规则的接入
- 它不等价于整机进入同一虚拟网段

## 5. 当前不应直接宣称的能力

如果没有额外证据，以下结论不应直接宣称：

- `tun` 等价于 TAP
- 仅凭 `tun` 配置即可自动建立完整三层互通
- `tun.bind` / `mask` 等价于一个已经落地的虚拟私网方案
- hub-spoke / full-mesh / 异地组网是当前应用原生内建且无需额外宿主机动作
- `cyfs-gateway` 可直接提供二层 bridge / switch 语义
- 不同网段设备会天然组成同一广播域
- 依赖 ARP、广播、mDNS、NetBIOS 的应用会自动按原生局域网方式工作
- `socks5` 方案等价于整机 VPN 或整机三层互通
- 不改宿主机路由 / 防火墙 / DNS 就一定能完成跨网段互通

这些都需要额外运行语义、实现细节或实测证据。

## 6. 当用户说“虚拟局域网”时的标准拆解

必须先拆成三个可能目标：

- 目标 A：让设备通过显式 IP、固定端口或静态域名互相访问
  典型特征：用户接受额外路由、防火墙或其他宿主机动作
- 目标 B：让支持代理的 app 访问远端服务
  典型特征：应用支持 SOCKS5 或系统代理，不要求整机进虚拟网段
- 目标 C：让设备像在同一交换网络中那样自动发现
  典型特征：依赖 ARP、广播、多播、mDNS、NetBIOS、零配置发现

在本 skill 当前知识边界内：

- 对目标 A，只能先检查 `tun` 的基础字段和外部前置条件，不能直接承诺“产品已原生打通”
- 对目标 B，可以按 `socks5` 代理接入方案分析
- 对目标 C，必须先声明 evidence 不足

## 7. 推荐回答口径

推荐这样表述：

- “当前 skill 只能直接确认 `tun` 是 IP 级 stack 配置模型。”
- “如果目标是不同网段设备通过固定地址互通，需要另外补足 underlay、路由、防火墙和实测验证。”
- “如果目标只是让支持代理的 app 访问远端服务，优先按 `socks5` 代理接入方案设计。”
- “如果目标要求完整广播域或零配置发现，当前 skill 没有足够证据证明 `cyfs-gateway` 原生支持，不能直接承诺。”

避免这样表述：

- “这就是一个完整虚拟局域网，和交换机效果一样”
- “任何局域网应用都能直接跑”
- “SOCKS5 等价于整机 VPN”
- “不用关心宿主机网络设置”
