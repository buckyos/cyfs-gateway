# 排障示例

## 示例 1：隧道建立了，但三层不通

### 用户问题

日志看起来 underlay 建好了，但 overlay IP 互 ping 不通。

### 推荐排查顺序

1. 检查 `tun.bind` 与 `mask` 是否属于同一规划
2. 检查目的流量是否真的走了 overlay
3. 检查宿主机防火墙是否放行 ICMP 或对应业务端口
4. 检查 MTU 是否异常

## 示例 2：三层通了，但业务不通

### 用户问题

能 ping 通，但业务连不上。

### 推荐排查顺序

1. 目标端口是否监听
2. 应用是否访问了 overlay 地址而不是物理地址
3. 应用是否依赖广播发现
4. 目标服务是否绑定到正确接口

## 示例 3：代理配置了，但应用不走代理

### 用户问题

SOCKS5 服务已经起来了，但客户端还是连不上远端。

### 推荐排查顺序

1. app 是否真的支持 SOCKS5 或系统代理
2. 系统代理是否被这个 app 继承
3. `target` 是否正确
4. `rule_config` 是否命中
5. `enable_tunnel` 是否按预期生效

## 示例 4：DNS 对了，但连接仍失败

### 用户问题

域名已经解析到预期地址，但 app 还是失败。

### 推荐排查顺序

1. 解析出的地址是否可达
2. 是否命中旧缓存
3. 流量是否进了 overlay 或代理
4. 应用协议本身是否允许跨网段 / 跨接口
