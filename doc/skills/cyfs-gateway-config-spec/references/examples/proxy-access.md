# Proxy Access 示例

## 示例 1：单个 App 通过 SOCKS5 访问远端服务

### 用户问题

我不需要整机进虚拟网段，只想让一个支持 SOCKS5 的客户端访问远端服务，怎么做？

### 目标归类

这是 `socks5` 型应用代理接入问题，不是 `tun` 的 IP 级配置问题。

### 推荐回答结构

1. 能力边界
2. socks server 的 `target`
3. 是否需要 `enable_tunnel`
4. app 侧如何配置 SOCKS5
5. 验证步骤

### 典型结论

- 这是“应用流量接入”，不是“整机网络接入”
- 只对支持代理的 app 生效
- 不要求分配 `tun` 地址

## 示例 2：多个支持代理的 App 访问异地资源

### 用户问题

几个客户端都支持系统代理，想访问异地网络资源，怎么分析？

### 目标归类

这是共享 `socks` 服务的代理接入方案。

### 推荐回答结构

- 哪些 app 走代理
- `target` 指向哪里
- `rule_config` 是否需要按域名 / 目标分类
- `enable_tunnel` 是否要显式开启某类 tunnel
- 哪些 app 不会被系统代理覆盖

## 示例 3：用户说“组局域网”，其实只是访问远端服务

### 用户问题

我想组个虚拟局域网，让浏览器和某个客户端能访问异地服务。

### 标准纠偏

如果需求只是让浏览器和特定客户端访问远端服务，而不是要讨论 `tun` 的地址规划和外部前置条件，优先按 `socks5` 代理接入方案分析。

## 示例 4：为什么 SOCKS5 不等于整机 VPN

### 用户问题

用了 socks server，是不是就等于整台机器已经进虚拟局域网了？

### 结论

不是。
当前有证据支持的是：`socks` server 可以让支持代理的应用通过代理访问目标服务，并可结合 `enable_tunnel` 与 `rule_config` 使用。
当前没有证据支持“整机所有流量天然进入同一虚拟网段”。
